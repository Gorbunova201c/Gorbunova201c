Структура Преимущества (+) Недостатки (-)
Список (List) ✓ Простота реализации
✓ Легко сериализовать (JSON, CSV)
✓ Минимальное использование памяти
✓ Подходит для хранения рёбер графа
✓ Гибкий размер
✓ Легко итерировать 		✗ Нет иерархической структуры
✗ O(n) время поиска связей
✗ Нужно хранить пары (родитель, ребёнок)
✗ Сложно восстановить дерево
✗ Нет прямого доступа к узлам
✗ Неудобно для навигации по дереву
Массив (Array) ✓ O(1) доступ по индексу
✓ Эффективное использование памяти
✓ Кэш-дружелюбный (contiguous memory)
✓ Предсказуемая структура памяти
✓ Быстрая навигация для полных деревьев
✓ Подходит для бинарных куч 		✗ Фиксированный размер (в некоторых ЯП)

Нужно управлять индексами вручную
✗ Сложно добавлять/удалять узлы
✗ Неинтуитивно для древовидной структуры
✗ Требует предварительного знания размера
Словарь (Dictionary) ✓ O(1) средний доступ по ключу
✓ Очень читаемый код
✓ Естественное представление дерева
✓ Легко добавлять/удалять узлы
✓ Идеально для JSON/API
✓ Не нужно управлять индексами 		✗ Больше накладных расходов памяти
✗ Возможны коллизии хешей
✗ Нет гарантий порядка (в старых ЯП)
✗ Чуть медленнее массива
✗ Ключи должны быть уникальными
✗ Хеш-функция требует вычислений
